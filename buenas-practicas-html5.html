Buenas prácticas HTML5

Redacta de forma limpia tu código HTML para que sea entendible.

Publicado por Miguel Zabala F. el 28 de Octubre, 2019

¿Qué son las buenas prácticas?

Se le conocen como buenas prácticas en el ámbito de la programación al buen uso de las herramientas que dichi lenguaje nos proporciona, para que dicho código realizado sea entendible y fácil de comprender para otra persona que le vaya hacer mantenimiento o mejoras. Un mal código genera tropiezos a la larga, ya sea para un programador, un diseñador o un equipo de trabajo.

En el mundo de la programación se debe entender que no todos los programadores realizamos el mismo código o el mismo flujo de pasos para dar soluciones o para crear.

"Entiende el problema o planifica antes de escribir el código."

Buenas prácticas en HTML5

A continuación te mostraremos qué buenas prácticas debes presentar a la hora de redactar tu primer código HTML:

Evitar estílos en líneas de código

Es muy frecuente que cuando comenzamos, muchos tutoriales nos enseñan a como cambiar le color del texto o tamaño de la fuente haciendo uso de los estilos dentro del index.html; esto se considera una mala práctica que hay que corregir desde el comienzo, ya que se puede llegar a converitr en una mala costumbre.

<p style="color: blue">Texto</p>
El texto que se encuentra en color rojo es el código que hay que evitar escribir dentro del código HTML, para eso tenemos una carpeta aparte llamada CSS donde se encuentran las hojas de estilo de la página.

Realizar comentarios tanto en CSS como HTML

Realizar comentarios en nuestro códigos ayuda a una mejor interpretación de él mismo, para que más adelante si necesitamos hacer cámbios o mejoras, se nos haga la tarea más fácil; también ayuda mucho cuando nos encontramos en un equipo de trabajo, así podemos entender mucho mejor el código de nuestro equipo y/o que funcion hace este código.

/*Así se hace un comentario en CSS*/

los códigos que encuentras en estos artículos pueden ser usados en tus propios archivos HTML o CSS.

Leer código de otros ayuda a una mejor interpretación de código a nivel profesional, nos enseñamos a nosotros mismos que el código puede ser escrito de muchas maneras, muchas veces de manera fácil como a la vez complejas.
